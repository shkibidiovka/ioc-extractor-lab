# REPORT.md

## 1. Что такое Git

Git - распределённая система контроля версий, которая позволяет отслеживать изменения в коде и других файлах проекта.  
Он хранит историю всех изменений в виде коммитов и позволяет вернуться к любой предыдущей версии.  
Git поддерживает работу с ветками, что даёт возможность разрабатывать новые функции параллельно, не ломая основную версию проекта.  
Также Git используется для совместной работы, код-ревью и контроля изменений в командной разработке.

## 4. Работа с .gitignore

В проекте были созданы файлы и директории, которые не должны попадать в репозиторий (служебные файлы, отчёты, переменные окружения).  
Для этого в файл `.gitignore` были добавлены правила, игнорирующие такие файлы, как `.env`, директория `outputs/`, а также `__pycache__`.  
Это предотвращает случайную публикацию временных файлов и потенциально чувствительных данных.

## 5. Инцидент с утечкой секрета (secrets.txt)

### Как была найдена утечка  
Секретный файл `secrets.txt` был намеренно добавлен в репозиторий и закоммичен.  
Для поиска утечки использовались команды:
- `git log -- secrets.txt`
- `git show`

Это позволило увидеть, в каком коммите файл появился и какое содержимое он содержал.

### Как была исправлена ошибка  
Файл `secrets.txt` был добавлен в `.gitignore`, после чего удалён из репозитория с помощью команды `git rm`, и изменения были закоммичены.

### Почему этого недостаточно  
Удаление файла из текущей версии репозитория не удаляет секрет из истории Git.  
Любой, кто имеет доступ к истории коммитов, всё ещё может восстановить файл и получить секрет.

### Что делают в реальной жизни  
В реальных проектах в такой ситуации выполняют переписывание истории репозитория с помощью `git filter-repo` 
После этого обязательно производится ротация скомпрометированного секрета, так как он считается утёкшим.
